# Andrey's Standup Style

## Format
- Plain text, no markdown formatting
- Each item on its own line with a **blank line between items**
- No emoji bullet points (unlike Antonio's style)
- No structured "What did you do / What will you do" sections (unlike the bot format)

## Voice
- Casual, lowercase start
- Brief descriptions with inline links to PRs/issues
- Conversational tone, not formal
- Links are just pasted inline, not formatted
- Use full GitHub URLs for PRs/issues (e.g. https://github.com/github/repo/pull/28) instead of shorthand like #28 — Slack will unfurl them nicely

## Example (from 2026-01-16)
```
catching up
pick up some tasks from https://github.com/org/repo/issues/1234
look over bobby's pr https://github.com/org/repo/pull/789
figure out calendar nonsense
```
